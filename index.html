Infrastructure Sizing Tool

Last updated: 02/27/2026  •  Interactive tool

Use this interactive sizing tool to determine the optimal infrastructure configuration for your Azure Virtual Desktop deployment. Configure your user count, workload profiles, graphics requirements, and storage preferences to receive tailored VM recommendations and estimated costs.

Why right-sizing matters

VM sizing directly affects both your cost and end-user experience. Oversized session hosts waste budget on unused capacity, while undersized hosts lead to CPU contention, slow logon times, and degraded application performance during peak loads. The goal is to right-size your VMs by matching compute resources to your workload requirements — factoring in whether you use personal or pooled host pools, expected user density, and the specific applications your users run. Microsoft recommends using a larger number of smaller VMs rather than a few large VMs, as they are easier to manage and provide more granular capacity control.

Scaling alongside sizing

Right-sizing determines the shape of your session hosts — scaling determines how many are running at any given time. Azure Virtual Desktop Autoscale scaling plans automatically adjust capacity based on user demand and schedules, powering off unused VMs during off-peak hours and spinning them up during ramp-up periods. Combining right-sized VMs with a well-tuned scaling plan — using breadth-first load balancing at ramp-up and depth-first during peak hours — helps organisations target 80–90% host pool utilisation while keeping costs in check. Most organisations align scaling schedules to user personas and working hours for maximum efficiency.

Tip

For mixed workload environments, select the profile that matches your heaviest user group, then adjust user counts accordingly. You can create multiple host pools with different VM sizes to optimise cost. Next steps

Now that you have your sizing recommendations, consider the following next steps to proceed with your deployment:

  • Review the complete prerequisites for Azure Virtual Desktop
  • Deploy Azure Virtual Desktop using the recommended VM configuration
  • Configure breadth-first load balancing to evenly distribute users across session hosts
  • Use the Azure Pricing Calculator for detailed cost analysis with your specific licensing
  • Configure identity and access management for your session hosts
  • Set up FSLogix profile containers for optimal user experience
  • Use performance counters to monitor User Input Delay (target <200ms) on session hosts
